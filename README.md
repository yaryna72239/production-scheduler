# 生產排程系統

專業的生產排程管理系統，支援日曆視圖、列表管理和統計報表。

## 功能特色

### 📅 日曆視圖
- 月曆檢視，一目了然
- 不同狀態以顏色區分（待產/生產中/已完成）
- 點擊日期快速查看當天工單

### 📋 工單管理
- 新增/編輯/刪除工單
- 工單編號、日期、產品、數量、產線、狀態、備註
- 支援搜尋功能

### 📊 統計報表
- 即時統計卡片（本月工單數、各狀態數量）
- 產線產能分佈圖
- 狀態分佈統計

### 🎨 介面設計
- 現代化 UI（Tailwind CSS）
- 響應式設計，支援手機/平板/桌機
- 直覺操作，無需教學

## 快速開始

### 1. 本地預覽
```bash
cd production-scheduler/public
python3 -m http.server 8080
```
然後開啟 http://localhost:8080

### 2. 部屬到 GitHub Pages

#### 建立 Repository
1. 前往 https://github.com/new
2. Repository name: `production-scheduler`
3. 選擇 Public
4. 點擊 Create repository

#### 上傳程式碼
```bash
cd /root/.openclaw/workspace/production-scheduler

# 初始化 git
git init
git add .
git commit -m "Initial commit"

# 連接 GitHub（請將 YOUR_USERNAME 換成你的 GitHub 帳號）
git remote add origin https://github.com/YOUR_USERNAME/production-scheduler.git
git branch -M main
git push -u origin main
```

#### 啟用 GitHub Pages
1. 進入 Repository → Settings → Pages
2. Source 選擇 "Deploy from a branch"
3. Branch 選擇 "main"，資料夾選 "/ (root)"
4. 點擊 Save

5. 等待幾分鐘後，網站將在 `https://YOUR_USERNAME.github.io/production-scheduler` 上線！

## 資料儲存說明

目前使用 **localStorage** 儲存資料（瀏覽器本地）：
- ✅ 免費，無需後端
- ✅ 即開即用
- ⚠️ 資料僅存在當前瀏覽器
- ⚠️ 清除瀏覽器資料會遺失

### 進階：連接雲端資料庫

如需跨裝置同步，可升級至 **Supabase**：

1. 註冊 https://supabase.com（免費）
2. 建立資料表：

```sql
create table production_orders (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  order_no text not null,
  date date not null,
  product_name text not null,
  quantity integer not null,
  production_line text not null,
  status text not null default 'pending',
  note text
);
```

3. 取得 API Key 並更新程式碼

## 使用說明

### 新增工單
1. 點擊右上角「新增工單」按鈕
2. 填寫工單資訊（編號、日期、產品、數量、產線、狀態）
3. 點擊「儲存」

### 查看排程
- **日曆視圖**：月曆模式，點擊日期查看當天工單
- **列表視圖**：表格模式，支援搜尋和排序
- **統計視圖**：圖表分析產能分佈

### 編輯/刪除
- 點擊日曆上的工單或列表中的編輯按鈕
- 刪除前會有確認提示
